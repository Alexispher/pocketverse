# PocketVerse Direct ROM Build

**PocketVerse** — это статический ретро-проект для GitHub Pages, который запускает игру формата **Game Boy `.gb`** прямо в браузере через веб-эмулятор.
Проект не требует сервера, базы данных, логина или установки программ на компьютер пользователя.

Игрок просто открывает страницу, нажимает кнопку запуска и играет прямо в браузере.

---

## 🎮 Ссылка для игры

Основная ссылка проекта:

```txt
https://alexispher.github.io/pocketverse/
```

Прямая ссылка для проверки ROM-файла:

```txt
https://alexispher.github.io/pocketverse/roms/game.gb
```

Если при открытии второй ссылки файл скачивается, значит ROM находится в правильной папке.
Если появляется ошибка **404**, значит файл расположен неправильно или имеет другое имя.

## 📌 Как работает проект

PocketVerse использует обычные веб-технологии:

* **HTML** — структура страницы;
* **CSS** — визуальный стиль ретро-консоли;
* **JavaScript** — запуск эмулятора, управление кнопками и логика интерфейса;
* **EmulatorJS** — браузерный эмулятор Game Boy;
* **GitHub Pages** — бесплатный статический хостинг.

Игра загружается напрямую по URL:

```txt
./roms/game.gb
```

После нажатия кнопки **Ligar console** JavaScript проверяет наличие файла `game.gb`, включает визуальную консоль, запускает EmulatorJS и передаёт ему путь к ROM-файлу.

Финальная структура репозитория должна быть такой:

```txt

├── style.css
├── app.js
├── README.md
└── roms/
    └── game.gb
```

Файл игры должен находиться именно здесь:

```txt
roms/game.gb
```

---

## ⚠️ Важные требования к ROM-файлу

Используйте только файл формата:

```txt
.gb
```

Не используйте:

```txt
.gba
.gbc
.nds
```

Файл должен быть переименован строго так:

```txt
game.gb
```

Не используйте пробелы, скобки, специальные символы или длинные названия, например:

```txt
Pokemon Red (U) [S][BF].gb
```

Такой вариант может вызвать проблемы при загрузке через URL.

Правильный вариант:

```txt
game.gb
```

## 🚀 Как запустить проект

1. Откройте репозиторий на GitHub.
2. Удалите старые файлы, если они остались от предыдущей версии.
3. Загрузите новые файлы:

```txt
index.html
style.css
app.js
README.md
```

4. Создайте папку:

```txt
roms
```

5. Поместите в неё файл:

```txt
game.gb
```

6. Перейдите в настройки репозитория:

```txt
Settings → Pages
```

7. Включите GitHub Pages для ветки `main`.
8. Откройте сайт:

```txt
https://alexispher.github.io/pocketverse/
```

## 🔧 Основные настройки в `app.js`

Главные параметры находятся в начале файла `app.js`:

```js
const ROM_URL = "./roms/game.gb";
const ROM_NAME = "Game";

const DATA_PATH = "https://cdn.emulatorjs.org/stable/data/";
const LOADER_PATH = "https://cdn.emulatorjs.org/stable/data/loader.js";
```

`ROM_URL` указывает, где находится файл игры.
`DATA_PATH` и `LOADER_PATH` указывают на стабильную CDN-версию EmulatorJS.

---

## 🌐 Почему используется CDN

В этой версии EmulatorJS загружается через CDN:

```txt
https://cdn.emulatorjs.org/stable/data/
```

Это сделано для того, чтобы избежать ошибок с неполной папкой `data`, неправильной загрузкой core-файлов или старым кэшем GitHub Pages.

Благодаря этому проекту не нужна локальная папка:

```txt
data/
```
